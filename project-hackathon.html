Hackathon · Penn State University · April 2026

RedScare
Hackathon Project

A period tracking and health insights web app built with one teammate in a 12-hour hackathon, focused on creating a more thoughtful and user-friendly approach to women's health.

HTML/EJS JavaScript Supabase Anthropic API Data Visualization

The Challenge

12 Hours to Ship Something Real

The hackathon prompt challenged participants to build something meaningful within a 12-hour window. My teammate and I zeroed in on a gap we both noticed: the existing tools for period and cycle tracking are either overly clinical, cluttered with advertising, or lack any real intelligence behind their insights.

We set out to build RedScare — a more thoughtful, user-centered approach to women's health tracking that combined clean UX with genuinely useful, AI-powered analysis.

What We Built

Features & Functionality

Cycle & Symptom Tracking

Log periods, symptoms, and mood over time with a clean, low-friction interface designed for daily use.

Insights Dashboard

Interactive data visualizations for spotting patterns and trends across cycles — built without a charting library.

AI-Powered Analysis

Integrated the Anthropic Claude API to generate personalized health insights based on the user's tracked data.

Real-Time Data

Supabase powers the backend — authentication, database, and real-time sync all in one layer.

My Contributions

What I Worked On

  • Co-architected the full-stack structure using EJS templating, CSS, and vanilla JavaScript on the frontend with Supabase handling auth and data persistence.
  • Built the interactive data visualization dashboard from scratch — translating raw cycle data into readable trend charts without relying on an external charting library.
  • Integrated the Anthropic Claude API to generate contextual, personalized health insights from the user's logged data.
  • Contributed to UX decisions and interface design throughout the build, keeping the experience calm and approachable given the sensitive subject matter.
  • Collaborated closely with my teammate under time pressure — scoping features, dividing work, and integrating our respective pieces into a cohesive, working product by demo time.

Outcome & Reflection

Shipping in 12 Hours

We successfully delivered a working, demo-ready application within the event window — cycle tracking, insights dashboard, AI analysis, and all. Building under a hard time constraint forced fast decision-making about what to scope in vs. cut, and sharpened my ability to stay execution-focused even when the problem space is broad.

Using the Anthropic API mid-hackathon was a highlight — seeing the model produce genuinely useful, contextual health insights from structured user data in real time validated the core idea quickly and added a layer of polish that stood out in the demo.
